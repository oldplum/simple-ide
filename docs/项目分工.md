太棒了！两个人组队开发是一个非常好的实战机会。双人协作最大的挑战不是代码写不出来，而是“如何避免互相踩脚（比如两个人同时修改同一个文件导致代码冲突）”。

根据你提供的详细需求文档 ，我们要把整个开发流程变成一条流水线，并采用“高内聚，低耦合”的原则来进行分工。这意味着你们俩负责的代码文件要尽量独立。

以下是为你们量身定制的开发流程与分工策略：

### 两人分工策略：双线并行开发

最好的分工方式是按照“主框架与文件系统（偏向广度）”**和**“核心编辑器与高亮算法（偏向深度）”来划分。

| 角色              | 核心职责 | 负责的模块与涉及类 |
| ----------------- | -------- | ------------------ |
| **开发者 A** <br> |

<br>(主框架与后勤) | 负责把软件的“壳子”搭起来，并打通系统底层的文件操作。 | <br>**主界面：** `QMainWindow`, `QTabWidget` (多标签页), 菜单栏, 状态栏 。

<br>

<br>**文件与会话：** `QFile`, `QFileDialog` (打开/保存/另存为), 编码检测, 未保存提示 。

<br>

<br>**系统配置：** QSS 主题切换, `QSettings` 记录上次打开的文件 。

 |
| **开发者 B** <br>

<br>(核心引擎与算法) | 负责把文字输入区做到极致，实现酷炫的彩色代码和打字体验。 | <br>**文本编辑区：** 继承 `QPlainTextEdit`，实现行号显示、自动缩进、括号匹配高亮 。

<br>

<br>**高亮引擎：** 继承 `QSyntaxHighlighter`，编写正则表达式进行逐行词法分析 。

<br>

**分工优势：** 开发者A主要都在写 `MainWindow.cpp` 和文件管理的类；开发者B主要都在写 `CodeEditor.cpp` 和 `SyntaxHighlighter.cpp`。你们俩极少会修改同一个文件，能最大程度避免 Git 冲突。

### 整体开发流程（时间顺序）

你们不能各写各的直到最后一天才合并，必须按照以下顺序**分阶段、迭代式**地推进。

**第一阶段：项目初始化与接口约定（两人一起做）**

* 创建之前说好的项目目录结构和 `.gitignore`。
* 在 Git 上建立仓库，两人都克隆到本地。
* **最关键的一步：** 你们俩要坐在一起，商量好 `CodeEditor` 这个自定义类需要暴露哪些公共函数给主界面调用。比如，开发者A的菜单栏里有一个“撤销”按钮，它点下去之后，是调用开发者B写的哪个接口。

**第二阶段：跑通核心骨架（并行开发）**

* 
**开发者 A：** 建立主窗口，放一个 `QTabWidget` 进去 。实现“新建文件”功能，能够成功往 `QTabWidget` 里动态添加一个新的页面。


* 
**开发者 B：** 建立 `CodeEditor` 类（暂时不加行号，能打字就行），并建立 `SyntaxHighlighter` 的空壳子，写几条简单的正则表达式（比如把所有的 "int" 和 "class" 变成蓝色）测试一下高亮引擎是否生效 。



**第三阶段：联调与打通血脉（第一次合并）**

* 将开发成果通过 Git 合并。
* 现在你们的程序应该能够：点击菜单栏“新建”，弹出一个标签页，里面包含开发者B写好的 `CodeEditor`，并且输入特定单词会变色。
* 开发者A顺势完成文件的“保存”和“打开”功能，把文本读入到编辑器中 。



**第四阶段：死磕进阶功能（并行开发）**

* 
**开发者 A：** 处理文件修改状态（标题栏加 `*` 号），实现关闭时的未保存提示 。开始编写 QSS 样式表，实现明亮和暗黑主题的切换 。


* 
**开发者 B：** 攻克最难的几个技术点：利用 `QTextBlock` 和绘制事件 (`paintEvent`) 把左侧行号画出来 ；重写键盘事件实现回车自动缩进；完善不同语言的高亮正则表达式 。



**第五阶段：最后的打磨与测试**

* 开发者A实现会话恢复（上次没关的文件下次启动自动打开）。


* 开发者B完成查找与替换的对话框逻辑 。


* 交叉测试：互相找对方模块的 Bug，查漏补缺。

### 电子猫咪互动系统（趣味功能）

**归属**：开发者 B（核心引擎与算法方向），与第四/第五阶段并行开发。

**新增文件**：
- `include/CatWidget.h` + `src/CatWidget.cpp` — 猫咪 UI 控件与情绪状态机
- `resources/cat_gifs/` — 存放不同情绪状态的 GIF 动图

**核心模块**：
1. **界面呈现**：`QLabel` + `QMovie` 播放 GIF 动图，悬浮于 `CodeEditor` 右下角或使用 `QDockWidget` 侧边栏
2. **情绪引擎**：有限状态机（FSM），状态包括默认/开心/生气/饥饿
3. **互动触发**：拦截 `CodeEditor` 的 `keyPressEvent` 信号，根据编码行为切换猫咪情绪

**与现有模块的接口**：
- `CodeEditor` 需新增信号（如 `bracketMatched()`、`massiveDelete()`）供 `CatWidget` 连接
- `MainWindow` 负责将 `CatWidget` 放入界面布局
